代码规范

  • 对于图文的简单组合显示，可以使用TextView的drawableLeft，drawableRight属性

流程规范

  • 接到需求后，如果是已有业务上的调整，要通过读代码熟悉业务，了解需求对已有业务的影响
    如果是新需求，通过交互图了解交互流程
  • 业务做完需要深度自测，把自己认为测试可能测不到的点测试一遍
  • 提交给测试的，应及时通知到位，并定期跟进测试结果，避免拖拉到上版本前出现大问题，影响上版本进度

    资源使用规范

  • string资源根据功能模块分类存放，常用的string检索项目，看是否可以直接用
  • 图片资源尺寸只需要xh.xxh,特殊情况除外
  • 对于项目中使用的色值有统一风格，路径Common\src\main\res\values\fcom_colors.xml，例外的直接在布局中写色值

    注释规范

  • Activity中布局变量写注释，方便阅读

    开发中需要使用的基类，工具类，控件等

  • 输入金额控件：ValidatorEditText
  • 输入银行卡每4位数新增空格控件：CardInputEditText

    其他你希望约定好需要放入规范中与别人一同协作的事或物

  • 每个人都可以做代码审查，发现比较明显的没按规范写的，可以提出来让对应人纠正

字符串资源命名规范

<项目模块>_<业务模块>_<资源描述>

项目模块

shop

  • 电商

    pay

  • 支付

    common

  • 公共

资源描述

资源描述部分，必须由小写字母，数字(不提倡)和_（下划线）组成，单词之间必须用下划线分隔，不得将多个单词和在一起！e.g.

// 正确    
common_sign_forgot_password // 忘记密码
// 错误
shop_product_gobuy // 去购物

迁移后的开发注意事项

  1. ToastUtil
  2. LazyOnClickListener
  3. IOSStyleToggle
  4. 格式化字符串的处理（String.format）
  5. DialogManager
  6. 字符串工具类（utils.string）下替代JudgmentLegal
  7. xml的tools命名空间
  8. live templete快捷键
  9. ApplicationContext
  10. FilePathManager
  11. ImageUtil
  12. HttpUtil
  13. MobileApi
  14. simpleXml的注解和@ReqParam
  15. BaseActivity：
    1. navigate系列
    2. toolbar系列
    3. setPageXXX系列
    4. initXXX系列
    5. regPresenter
  16. MVP框架
    1. IBaseCoreView/IBaseUIView/IListView
    2. presenter 调用Model；存储控制标记，页面数据；onDestory;onSuccess/onFailure
    3. model 构造函数注册接口
  17. 图片上传套件UploadFileView
  18. CommonLibrary是汇旺和三维度项目共用，需要专门维护，只要改了CommonLibrary及以下的module里的代码就必须:
    1. 写rb/rl（live templete快捷键）注释
    2. 文件头写修改记录
    3. 告诉大家
    4. 建议提交前先知会本人
  19. DevTool
  20. 尽最大可能不要再修改Constant/URL/GlobalVariable
  21. AppInfoUtil
  22. UserManager
  23. MoneyFormatter/StringFormatter/StringValidator/StringConverter
  24. PullableLayout和RecyclerView
  25. RecyclerBanner
  26. BaseRecyclerAdapter
  27. SimpleImageAdapter
  28. 禁止使用废弃类和废弃属性
  29. 文件头和修改注释
  30. BaseFragment 和 v4 Fragment
  31. StaticFragmentPager 静态的Fragment的ViewPager
  32. BasePullableActivity 和 BasePullableFragment
  33. layout 里 控件id 必须按照格式书写 e.g. btn_sign_in_submit;对应java代码里控件的变量名必须与控件id一致
